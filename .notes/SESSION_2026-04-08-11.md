## Timeline of key experiments

### Day 1 (April 8) — Backpropagation proposal + initial walker research

**Started with**: Andy proposed localized backpropagation via Teacher
Forcing — a gradient delta layer on top of the frozen graph.

**Decided against it** because the walker's grammar issues come from beam
search scoring, not from missing edges. The edges already exist; the
walker just doesn't follow them because PMI pull drowns the grammatical
signal.

**Instead explored**: Fix 1 (log-compressed pull), Fix 2 (harder trigram
gate), Fix 3 (grammar-first fenced walker), Fix 4 (cadence walker),
Fix 4b (cadence + bridge search). All documented in WALKER_RESEARCH.md.

**Key finding**: grammar and semantics are separable structures in the
same graph topology. The same graph encodes both, extractable through
different walker configurations.

### Day 2 (April 9 morning) — v14 model comparison + path coherence hypothesis

**Tested v14 (Gutenberg-only)** against v16 (blend). v14 was worse on
every prompt — fewer edges, narrative bias, Spanish token artifacts.
Confirmed v16 blend is the right model.

**Andy's hypothesis**: "how is the next token connected to the previous
tokens of the total path? That matters too." This became the path
memory experiment.

### Day 2 (April 9 midday) — Path memory experiments

**Experiment: minimal walker (pure weight, no targets).**
Result: every prompt collapsed to "the same time , and the same time."
Pure greedy max-weight is NOT grammar — it's the language's center of
gravity. **The shipped walker's target-seeking is the mechanism that
breaks out of the max-weight attractor.**

**Experiment: target walker + path memory (v2).**
Added `full_path_window` to beam search. Content-only memory on both
sides (function words excluded). Multiplicative nudge on content
candidates.

Result: Dark clouds 3→13 tokens, Army marched 7→20, Fire burned 14→7
(REGRESSION). Memory rescues sinks but couldn't beat v1 on all prompts.

### Day 2 (April 9 afternoon) — Rank-based filters + topology measurements

**Discovered**: `in_degree > 15000` filter was silently excluding
"discovered" (in_deg 15990, rank 628). Built VocabRank with 2-tier
rank-based classification.

**Measured vocab distribution** (`measure_vocab_ranks.py`): the natural
knee in the in_degree curve is at rank ~150. Content words from our
prompts sit at ranks 600-5000.

**Found**: no single `n_semi_function` cutoff works for both default and
weather prompts. Default prompts prefer 700 (fewer semi-common targets),
weather prefers 500 (more targets available in sparse fields). **First
concrete proof that adaptive config is needed.**

**Andy's correction**: "activate() operates on the STATIC full graph
where the in_degree distribution is known and fixed. The rank-based
fix belongs DOWNSTREAM of activation." Reverted activate_v2, kept
rank fix only in find_targets and density calculation.

### Day 2 (April 9 evening) — The breakthrough: sentence anatomy

**Andy's directive**: "Before we keep iterating the code, not knowing
where to go, I propose to learn more about the topology of the
network. What does a sentence actually look like? Maybe we can trace
actual sentences from the corpus."

**Built `sentence_anatomy.py`**. Sampled 545 sentences from all three
corpus files. Measured per-position: in_degree, out_degree, pr_ratio,
forward weight, trigram validity.

**Discovery**: real sentences have a corpus-invariant topological wave:
- Rank 105-140 mid-sentence (not 24-81 like v1)
- Forward weight 700K-1.5M (not 6-9M like v1)
- Trigram coverage 65-72% (not 83-100% like v1)
- pr_ratio 0.03-0.05 (not 0.01 like v1)

**v1 was more extreme than real English on every measurable axis.**
The obvious approach (maximize weight) was wrong. The correct approach
(match the measured wave) was counter-intuitive.

## Key findings (ordered by importance)

### 1. The topological wave (biggest discovery)

Real English sentences have a measurable, corpus-invariant topological
signature. Rank oscillates 105-140, forward weight stays 700K-1.5M,
trigram coverage ~65-72%. This is a property of ENGLISH, not of the
corpus. The wave is the grammar.

### 2. Maximization is wrong

The weight-maximizing walker was more extreme than real English on every
axis. The path to better output was not stronger grammar enforcement or
heavier semantic pull — it was matching the measured wave. Counter-
intuitive finding.

### 3. Grammar and semantics are separable

Same graph encodes both. Forward edges + trigrams = grammar. PMI = 
semantics. Walker configured for one produces strong results on that
dimension while failing on the other. Combining them remains the open
problem.

### 4. Absolute thresholds are harmful

Five different `in_degree` thresholds, none justified, none consistent.
Silently broke on edge cases ("discovered" filtered at 15990). Rank-based
classification is model-agnostic but no single cutoff works for all
prompts — proves adaptive config is needed.

### 5. The shipped walker's target-seeking is essential

Pure weight-following collapses to "the same time, and the same time."
The target-seeking architecture (activate → find_targets → walk_to_target)
is not an overlay — it's the mechanism that breaks out of the max-weight
attractor.

### 6. Path memory works as a rescue signal

On sink-dominated prompts (Dark clouds, The fire burned), memory extends
output by 50-500%. On working prompts, identical output (no regression
when not needed). Content-only on both sides is critical — function word
connections are noise.

### 7. No single fixed configuration works for all prompts

Different subnetwork topologies need different parameters. Default
prompts prefer tighter target filtering, weather prompts prefer looser.
Adaptive config (derive parameters from subnetwork measurements) is the
required next step.

## Next session priorities (in order)

1. **Sentence anatomy on the sentence STRUCTURE level** — not just per-token
   features, but clause patterns. Where do verbs appear relative to nouns?
   Where do commas mark clause boundaries? Can we detect "I need a verb here"
   from the topology?

2. **Two-pass architecture** — content skeleton first (what to say), then
   grammatical infill (how to say it). Currently both happen simultaneously,
   which is why phrases are coherent but sentences aren't.

3. **Adaptive config from subnetwork** — derive memory_alpha, n_semi_function,
   and other coefficients from the activated subnetwork's measurable properties.
   Proven needed by the defaults-vs-weather tradeoff.

4. **Register locking** — per-corpus profiles (gutenberg archaic, fineweb
   factual, openwebtext casual). Select based on prompt PMI signature. Prevents
   mid-sentence register switching.
